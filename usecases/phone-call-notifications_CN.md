# 电话通知

你的智能体已经为你监控各种内容 —— 股票、电子邮件、智能家居、日历 —— 但通知很容易被忽略。推送通知堆积成山。聊天消息被埋没。对于真正重要的事情，你需要一个无法随意滑走的东西。

这个使用案例为你的智能体提供了一个电话通知渠道。当某些事情足够紧急时，智能体会拨打你的真实电话号码，告诉你发生了什么，你还可以回应。这是双向对话，而不是自动拨打的电话营销。

## 功能说明

- 智能体判断某些事情值得你关注（价格警报、紧急邮件、预约提醒，任何内容）
- 智能体通过 [clawr.ing](https://clawr.ing) 拨打你的电话，这是一项托管式电话服务 —— 无需 Twilio 设置，无需配置 API 密钥
- 你接听电话，听到警报，可以实时询问后续问题
- 对话结束后电话挂断

核心思想：智能体打电话给你。你不用打电话给智能体。这适用于心跳检查、cron 作业或任何触发器 —— 智能体评估某些事情是否值得打电话，然后采取行动。

## 为什么有效

OpenClaw 智能体已经具有主动性 —— 心跳、cron、事件触发。但它们的输出渠道仅限于你可能不会检查的聊天平台。电话是唯一能可靠引起你注意的通知渠道，特别是当你远离办公桌时。

clawr.ing 处理电话基础设施。你粘贴一个设置提示词，智能体就获得了打电话的能力。无需 Twilio 账户，无需电话号码配置，无需 webhook 配置。该服务在 100+ 个国家提供真实的 PSTN 电话（不是 VoIP 叠加层）。

## 所需技能

- [clawr.ing](https://clawhub.ai/marcospgp/clawring) — 通过将 [clawr.ing 仪表板](https://clawr.ing) 中的设置提示词粘贴到你的 OpenClaw 聊天中来安装。无需 CLI 安装。

就这样。没有其他依赖项。设置提示词包含 API 密钥和技能文档链接 —— 智能体会阅读它们并弄清楚其余部分。

## 示例：早间简报电话

设置一个 cron 作业，每天早上给你打电话，提供个性化简报：

```
每个工作日上午 7:30，打电话给我进行早间简报。包括：
- 我所在城市的天气预报
- 我今天的日历安排
- 任何昨晚收到的紧急邮件
- 与我兴趣相关的前 3 条新闻头条

保持简洁 —— 目标是 2 分钟以内。如果我问问题，请回答。
如果我不接听，不要重试。
```

## 示例：价格警报

告诉智能体要监控什么以及何时打电话：

```
监控 NVDA 股票价格。如果它在一天内下跌超过 5%，
立即打电话给我，告诉我发生了什么。包括任何可能
解释下跌的相关新闻。
```

## 示例：紧急邮件过滤器

```
在工作时间内，每 15 分钟检查一次我的收件箱。
如果你看到来自我老板的电子邮件或任何标记为紧急的电子邮件，
打电话给我并提供摘要。对于其他所有内容，只需发送聊天消息。
```

## 核心见解

- **不要过度使用。** 整个要点是电话意味着"这真的很重要"。如果你的智能体一天给你打 10 次电话，你会开始忽略它。为值得打电话的内容和值得聊天的内容设定明确的阈值。
- **与心跳或 cron 配对。** clawr.ing 是传递渠道 —— 你仍然需要触发器。心跳检查（每 30 分钟）适用于监控任务。Cron 作业适用于定时简报。
- **双向对话。** 与推送通知不同，你可以在电话上询问后续问题。"等等，哪封邮件？"或"现在的价格是多少？" —— 智能体实时响应。
- **快速模型用于通话。** 电话对话需要快速响应。如果你的 OpenClaw 支持技能级别的模型路由，请将 clawr.ing 分配给快速模型（Haiku 级别）。思考声音可以覆盖延迟，但越快越好。
- **隐私。** clawr.ing 不存储录音或转录。音频在传输过程中加密，处理后被丢弃。
