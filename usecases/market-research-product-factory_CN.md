# 市场研究与产品工厂

你想构建产品但不知道构建什么。或者你有业务，需要了解客户在什么方面挣扎。这个工作流程使用 Last 30 Days 技能从 Reddit 和 X 中挖掘真正的痛点，然后让 OpenClaw 构建解决这些问题的解决方案。

## 功能说明

- 使用 [Last 30 Days](https://github.com/mvanhorn/last30days-skill/) 技能在过去 30 天内研究 Reddit 和 X 上的任何主题
- 揭示人们发布的真正挑战、抱怨和功能请求
- 帮助你从真正的用户痛点识别产品机会
- 更进一步：让 OpenClaw 构建解决其中一个挑战的 MVP
- 创建一个从研究到产品的完整流水线，无需你编写任何代码

## 痛点

大多数有抱负的创业者都在"构建什么"问题上挣扎。传统市场研究意味着手动浏览论坛、社交媒体和评论网站数小时。这自动化了整个从发现到原型的流水线。

## 所需技能

- Matt Van Horde 的 [Last 30 Days](https://github.com/mvanhorn/last30days-skill/) 技能
- Telegram 或 Discord 集成用于接收研究报告

## 如何设置

1. 安装 Last 30 Days 技能：
```text
安装此技能：https://github.com/mvanhorn/last30days-skill/
```

2. 对任何主题进行研究：
```text
请使用 Last 30 Days 技能来研究人们在 [你的主题] 方面
遇到的挑战。

组织调查结果为：
- 顶级痛点（按频率排名）
- 具体抱怨和功能请求
- 现有解决方案的差距
- 新产品的机会
```

3. 选择一个痛点并让 OpenClaw 构建解决方案：
```text
为我构建一个解决 [研究中的痛点] 的 MVP。
保持简单 —— 仅核心功能。
将其作为我可以与他人分享的 Web 应用程序发布。
```

4. 为了持续的市场情报，安排定期研究：
```text
每周一早上，使用 Last 30 Days 技能来研究人们在 Reddit 和 X 上关于 [你的细分市场] 的说法。总结顶级机会并将它们发送到我的 Telegram。
```

## 真实世界示例

```text
"使用 Last 30 Days 技能来研究人们在 OpenClaw 方面遇到的挑战。"

结果：
- 设置困难：许多用户在初始配置方面遇到困难
- 技能发现：人们找不到能完成他们所需工作的技能
- 成本担忧：用户想要更便宜的模型替代品

 "为我构建一个简单的 Web 应用程序，通过引导向导使 OpenClaw 设置更容易。"

OpenClaw 构建应用程序。你发布它。你就有了一个产品。
```

## 核心见解

- 这是**自动驾驶创业**：发现问题 → 验证需求 → 构建解决方案，全部通过短信完成。
- Last 30 Days 技能给你真实的、未经审查的用户情绪 —— 不是经过净化的调查数据。
- 你不需要技术。OpenClaw 进行研究**和**构建。
- 安排每周研究以保持了解你市场中不断演变的痛点。
