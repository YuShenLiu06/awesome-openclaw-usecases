# 自动会议笔记与行动项

你刚刚结束了一个 45 分钟的团队电话。现在你需要编写摘要、提取行动项并将它们分发到 Jira、Linear 或 Todoist —— 手动。等你完成时，下次会议已经开始了。如果你的智能体处理所有这一切，转录落地时会发生什么？

这个使用案例将任何会议转录转化为结构化笔记，并自动在你的项目管理工具中创建任务。

## 痛点

会议笔记很繁琐但很关键。大多数人要么跳过它们（并丢失上下文），要么花 20+ 分钟编写它们。行动项被遗忘，因为它们存在于某人的脑海中或埋没在聊天线程中。这个智能体消除了"我们讨论了它"和"它被跟踪和分配"之间的差距。

## 功能说明

- **监视**新的会议转录（通过 Otter.ai 导出、Google Meet 转录、Zoom 录制摘要或简单粘贴到聊天中）
- **提取**关键决策、讨论主题和带有所有者和截止日期的行动项
- **在 Jira、Linear、Todoist 或 Notion 中创建任务** —— 分配给正确的人并带有会议中的上下文
- **将摘要发布**到 Slack 或 Discord，以便整个团队都有记录
- **跟进** —— 可选地在截止日期前通过计划提醒通知被分配者

## 所需技能

- Jira、Linear、Todoist 或 Notion 集成（用于任务创建）
- Slack 或 Discord 集成（用于发布摘要）
- 文件系统访问（用于阅读转录文件）
- 计划 / cron（用于跟进提醒）
- 可选：Otter.ai、Fireflies.ai 或 Google Meet API 用于自动转录检索

## 如何设置

1. 选择你的转录来源。最简单的方法是直接将转录粘贴到聊天中。为了自动化，设置文件夹监视或 API 集成。

2. 提示 OpenClaw：
```text
我刚刚结束了一个会议。以下是转录：

[在此粘贴转录或指向文件]

请：
1. 编写一个简洁的摘要（最多 5 个要点），涵盖关键决策和主题。
2. 提取所有行动项。对于每个行动项，识别：
   - 需要做什么
   - 谁负责（将姓名匹配到我的团队）
   - 截止日期（如果提到了，否则标记为"TBD"）
3. 为每个行动项在 Jira 中创建一个工单，分配给正确的人。
4. 将完整摘要发布到 Slack 中的 #meeting-notes。
```

3. 对于完全自动化流水线（转录文件夹监视）：
```text
设置一个循环任务：每 30 分钟，检查 ~/meeting-transcripts/ 以查找
新的 .txt 或 .vtt 文件。当你找到一个时：

1. 将转录解析为带有行动项的结构化摘要。
2. 为每个行动项在 Linear 中创建任务。
3. 将摘要发布到 Slack 中的 #team-updates。
4. 将处理后的文件移动到 ~/meeting-transcripts/processed/。

对于每个有截止日期的行动项，设置提醒以便在截止日期前一天
在 Slack 中通知被分配者。
```

4. 自定义输出格式：
```text
编写会议摘要时，始终使用此结构：
- **日期和与会者**在顶部
- **关键决策** —— 决定内容的编号列表
- **行动项** —— 带有列的表格：任务、所有者、截止日期、状态
- **开放问题** —— 任何需要跟进的未解决问题
```

## 核心见解

- 真正的价值不在于摘要 —— 它在于**自动任务创建**。不会变成跟踪任务的会议笔记只是文档剧场。
- 将此与[Todoist 任务管理器](todoist-task-manager.md)使用案例配对，以获得智能体创建任务的完全可见性。
- 来自 Zoom 或 Google Meet 的 VTT/SRT 字幕文件作为输入很有效 —— 它们包含时间戳，这有助于智能体将陈述归因于发言人。
- 开始简单（粘贴转录，获取摘要）并增量自动化。在验证输出质量之前不要过度设计流水线。
