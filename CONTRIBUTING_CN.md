# 贡献指南

## 添加使用案例

1. 在 `/usecases` 文件夹中创建一个 markdown 文件，以你的使用案例命名。
2. 详细添加你的使用案例。以下是建议涵盖的几个要点：
   - **痛点** — 这解决了什么问题？
   - **功能说明** — 使用案例的简要解释
   - **提示词** — 你给 OpenClaw 的实际提示词/配置
   - **所需技能** — 需要哪些 OpenClaw 技能
   - **相关链接** — 相关文档、API、工具
3. 在主 [README.md](README.md) 的相关类别表格中添加一行。
4. 提交 PR

## 指南

- 每个 markdown 文件一个使用案例
- 保持描述简洁，但足以让他人复现
- 如果你的使用案例不符合现有类别，请在你的 PR 中建议一个新类别
- 如果方法有明显差异，重复的案例是可以的
- 不要使用 AI 来构思新的使用案例。只建议你已经实际测试并验证过的使用案例。
- **不接受与加密货币相关的使用案例** — 这些将不会被接受。
